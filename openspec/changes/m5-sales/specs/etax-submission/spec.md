## ADDED Requirements

### Requirement: e-Tax XML submission
`POST /api/v1/etax/{invoice_id}/submit` (perm `sales.etax.submit`) SHALL enqueue an
asynchronous job that produces an RD e-Tax XML document for the invoice and respond **202**
with a `job_id`. The produced XML SHALL be flagged **non-authoritative** — it is a generated
representation, not a confirmed Revenue Department submission.

#### Scenario: e-Tax submission is accepted asynchronously
- **WHEN** e-Tax submission is requested for an invoice
- **THEN** the request is accepted with 202 and a `job_id`, and an RD e-Tax XML document is generated by the job

#### Scenario: Generated e-Tax XML is non-authoritative
- **WHEN** the e-Tax XML is produced
- **THEN** it is flagged non-authoritative rather than presented as a confirmed RD submission
